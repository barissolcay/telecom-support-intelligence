# Security policy

Report vulnerabilities privately through GitHub Security Advisories. Do not include customer data or secrets in a report.

TelcoAssist is an educational prototype and must not be connected to production customer systems without a separate threat model, access-control review, data protection impact assessment, penetration test, and operator approval. Raw PII must never be added to issues, logs, fixtures, screenshots, or repository history.

Supported security controls include server-side secret handling, input limits, PII redaction, active-source filtering, grounded refusal, prompt/data separation, and human approval before outbound responses.
